# AI 长期代理协议

一个用于AI代理在多个上下文窗口间工作的综合协议。

## 解决的问题

| 问题 | 说明 |
|------|------|
| 一次性做太多 | AI试图一次完成所有事情，留下半成品 |
| 过早宣布完成 | AI未经测试就宣布项目完成 |

---

## 快速入门

### 本地开发用户

| 步骤 | 操作 |
|------|------|
| 首次会话 | 将 `AGENT_PROTOCOL.md` 拖给AI + 发送 `请阅读并遵循本项目的AGENT_PROTOCOL.md文件` + 项目需求 |
| 后续会话 | 发送 `请阅读并遵循本项目的AGENT_PROTOCOL.md文件` |

### 网页/移动端用户

| 步骤 | 操作 |
|------|------|
| 首次会话 | 将 `AGENT_PROTOCOL.md` 拖给AI + 发送 `请阅读并遵循本项目的AGENT_PROTOCOL.md文件` + 项目需求 |
| Token > 70% | 发送触发词：`会话总结` / `交接` / `summary` / `handover` |
| 新会话 | 粘贴交接总结 + 发送 `请阅读并遵循本项目的AGENT_PROTOCOL.md文件` |

## 核心规则

| 规则 | 说明 |
|------|------|
| 单一功能 | 每次会话只处理一个功能 |
| 干净状态 | 会话结束前代码必须能运行 |
| 端到端测试 | 必须实际测试，不仅仅是代码审查 |
| 记录进度 | 始终更新进度或生成交接总结 |
